# 基于 YOLO 模型的桃子成熟度分拣系统 —— 实训报告

## 一、项目概述

本项目为华清远见5天高校实训课题，目标是构建一套**基于 YOLO 目标检测模型
的桃子成熟度分拣系统**。系统使用 PyQt5 与定时器模拟工业生产线传送带，周期性
采集图像并预处理，送入经过微调训练的 YOLO 模型识别**桃子的成熟度**
（未成熟/半成熟/成熟，即 raw/half-ripe/ripe），再依据识别结果给出分级去向
并实时统计，完整复现工业视觉分级的工作流程。

## 二、体验目标

1. 掌握 YOLO 目标检测模型从数据标注到部署的完整使用流程；
2. 掌握使用 LabelImg 制作 YOLO 数据集、划分训练/验证集；
3. 掌握 ultralytics YOLO 的训练、调优、评估、预测；
4. 掌握 OpenCV 图像获取与预处理、NumPy 数组操作；
5. 掌握 PyQt5 桌面界面开发与定时器驱动的产线仿真；
6. 完成系统联调，形成可运行的端到端项目。

## 三、系统结构

```
        ┌─────────────┐    定时节拍(QTimer)    ┌──────────────┐
        │ 传送带图像源 │ ─────────────────────▶ │  图像预处理   │
        │(相机/文件夹/ │                        │ (OpenCV去噪) │
        │  合成画面)   │                        └──────┬───────┘
        └─────────────┘                               │
                                                       ▼
        ┌─────────────┐    分级结果/料道/统计   ┌──────────────┐
        │  PyQt5 界面  │ ◀───────────────────── │  YOLO 模型    │
        │ (实时显示)   │                        │ (best.pt 预测)│
        └─────────────┘                        └──────────────┘
```

## 四、技术点

| 模块 | 技术 | 说明 |
|------|------|------|
| 数据 | LabelImg / 合成生成 | YOLO 格式标注：`cls cx cy w h`(归一化)，cls∈{0 raw,1 half-ripe,2 ripe} |
| 模型 | YOLOv8n (ultralytics) | 以预训练权重微调，nano 版 CPU 可训 |
| 图像 | OpenCV + NumPy | 取帧、高斯去噪、颜色空间转换、画框 |
| 界面 | PyQt5 + QTimer | 传送带节拍、结果展示、统计与日志 |

## 五、关键实现说明

### 5.1 数据集（第2天）
`scripts/01_make_dataset.py` 生成带传送带背景、随机数量桃子的合成图像，
按成熟度画出青/黄绿/红三色果身并叠加表皮色斑，输出 YOLO 标注与 `data.yaml`。
真实场景可用手机拍照 + LabelImg 标注(对照 classes.txt 的 raw/half-ripe/ripe)，
再用 `02_split_dataset.py` 按 8:2 划分训练/验证集。

### 5.2 训练与调优（第2天）
`scripts/03_train.py` 调用 `YOLO(yolov8n.pt).train(...)`，支持 epochs、
imgsz、batch、device 参数，并启用早停(patience=20)。最佳权重输出至
`runs/detect/fruit_sorter/weights/best.pt`。

### 5.3 评估（第3天）
`scripts/04_eval.py` 通过 `model.val()` 输出 mAP50、mAP50-95、各类
Precision/Recall，并自动保存混淆矩阵与 PR 曲线。

### 5.4 预测（第2-3天）
`scripts/05_predict.py` 对图片或文件夹推理并保存可视化结果。

### 5.5 产线集成（第4-5天）
`src/main_gui.py` 用 `QTimer` 周期触发 `on_tick()`：取帧 → 预处理 →
`FruitDetector.predict()` → 画框显示 → 更新料道与统计。`src/detector.py`
封装推理并把英文类别映射为中文名与分级料道(未熟道/半熟道/成熟道)；
`src/conveyor.py` 提供“文件夹轮播 / 实时合成”两种图像源以模拟相机抓帧。

## 六、运行步骤

见 `README.md` 第三节“快速开始”，或执行 `bash run_demo.sh` 一键演示。

## 七、结果与结论

系统实现了从数据准备、模型训练评估到产线界面集成的完整闭环。合成数据集
生成、数据划分、传送带取帧与预处理已在 numpy/opencv 环境验证通过；训练与
图形界面在安装 ultralytics、torch、PyQt5 后即可运行。通过本实训，掌握了
YOLO 目标检测的工程化落地方法与桌面端工业视觉应用的开发模式。

## 八、改进方向

1. 用真实桃子照片扩充数据，提升泛化能力（合成"卡通桃子"存在域差异）；
2. 接入真实工业相机(USB/GigE)替换合成/文件夹图像源；
3. 细化成熟度等级（如增加"过熟/腐烂"类）或叠加缺陷检测、计数与速度统计；
4. 用 YOLOv8s/m 提升精度，或导出 ONNX/TensorRT 加速部署；
5. 增加分级执行机构(舵机/气缸)的串口控制对接。
