# 真实数据训练全流程教程

> 目标：让桃子成熟度分拣系统在**真实照片**上也能准确区分
> 未成熟(raw) / 半成熟(half-ripe) / 成熟(ripe)，彻底解决"合成图能认、
> 真实图认不准、半熟和生/熟分不清"的问题。
>
> 适用环境：Mac(Apple 芯片) / Windows·Linux(NVIDIA 显卡) / 纯 CPU 机器。
> 全程跟着复制命令即可，无需深入懂 YOLO。

---

## 〇、为什么要做这件事（先理解，再动手）

原项目用 `scripts/01_make_dataset.py` 生成的是 OpenCV **画出来的"卡通桃子"**：
青桃是一个绿色圆、半熟是黄绿圆、熟是红色圆。模型据此学到的成熟度判别依据，
**本质上只是几个固定的颜色像素值**，并没有学到真实世界判断成熟度的特征——
底色由绿转黄的过渡、红晕(blush)的分布与面积、表皮斑点与光照不均。

于是出现你看到的现象：

- 合成图 100% 识别正确（测试图和训练图是同一种"画法"）；
- 真实照片容易出错，尤其**半成熟(half-ripe)这个中间态**最难判，常被划到
  raw 或 ripe 两边。

这在机器学习里叫 **domain gap（域差异）**。**唯一的根治办法是用真实图片训练。**

> ⚠️ 注意：本任务识别的是**同一种水果(桃子)的成熟阶段**，不是不同水果。
> ultralytics 官方预训练模型(含 Open Images V7)只有笼统的 "Peach" 类、
> **没有成熟度细分标签**，无法直接用于本任务——所以**没有"零训练即用"的捷径**，
> 真实数据必须用带成熟度标注的数据集，或自己标注。
>
> （项目里残留的 `scripts/06_fetch_real_dataset.py`、`08_pretrained_oiv7.py`
> 是旧"多水果分拣"方向的产物，已标注废弃，对成熟度任务无效，请勿使用。）

## 一、环境准备（只需做一次）

### 1.1 确认 Python

需要 Python 3.8 ~ 3.11。终端执行：

```bash
python3 --version
```

### 1.2 进入项目目录

```bash
cd ~/Desktop/fruit-sorting-yolo
```

> 下文所有命令都默认在这个目录下执行。

### 1.3 创建并激活虚拟环境（推荐，隔离依赖）

**Mac / Linux：**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Windows（PowerShell）：**

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

激活成功后，命令行前面会出现 `(.venv)` 字样。

### 1.4 安装依赖

```bash
pip install -U pip
pip install -r requirements.txt      # ultralytics / opencv / numpy / PyQt5 / pyyaml
```

> 国内网络慢可加清华镜像：
> `pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`

### 1.5 验证安装

```bash
python -c "import ultralytics, cv2; print('环境 OK')"
```

打印 `环境 OK` 即表示就绪。

## 二、获取带成熟度标注的真实数据集

成熟度数据没有"一条命令自动下载"的官方源，下面给两条可行路径，**任选其一或
两者结合**。核心都是得到 YOLO 格式：图片 + 同名 `.txt` 标注 + `data.yaml`。

> 类别顺序务必与本项目一致：**0=raw  1=half-ripe  2=ripe**。

### 路径 A：从 Roboflow Universe 下载现成数据集（最省事）

[Roboflow Universe](https://universe.roboflow.com/) 上有多个公开的
"peach ripeness / peach maturity / peach ripening" 目标检测数据集，已按成熟度
分类并带边界框。

1. 在 Roboflow Universe 搜索 `peach ripeness`，挑一个类别贴合(raw/half-ripe/
   ripe，或 unripe/semi-ripe/ripe 等近义)、图片量合适的数据集；
2. 进入数据集 → **Download** → 格式选 **YOLOv8** → 下载 zip 并解压到项目下，
   例如 `dataset_real/`；
3. 解压后是这样的结构（Roboflow 的标准导出）：

   ```
   dataset_real/
   ├── train/images/*.jpg   train/labels/*.txt
   ├── valid/images/*.jpg   valid/labels/*.txt
   ├── test/ ...
   └── data.yaml            # 里面已含 names 列表
   ```

4. **核对并对齐类别**：打开 `dataset_real/data.yaml`，看 `names` 是否为
   `['raw','half-ripe','ripe']` 且顺序一致。
   - 若名字不同但含义对应（如 `unripe/half_ripe/ripe`）：可直接沿用它的名字
     训练，只要**三类的语义和你预期一致**即可；GUI 显示名以
     `src/detector.py` 的 `CN_NAME` 为准，必要时把 `CN_NAME`/`SORT_BIN`/
     `BOX_COLOR` 的键改成数据集实际用的英文名。
   - 若顺序不同：YOLO 只认 id，**务必让 `data.yaml` 的 names 顺序与标注里的
     id 对应**（Roboflow 导出的 yaml 已经是对的，不要手动打乱）。
5. Roboflow 的 yaml 里 `train/val` 路径可能是相对路径，确认能被找到；
   也可手动改成绝对路径。

### 路径 B：自己拍照 + LabelImg 标注（数据最贴合你的场景）

1. 拍摄不同成熟度的桃子照片：青桃(raw)、转色中/半红(half-ripe)、成熟桃(ripe)，
   每类尽量几十张以上，放入 `dataset_raw/`；
2. 安装并打开标注工具：`pip install labelImg` → 运行 `labelImg`；
3. 在 LabelImg 里：**格式切到 YOLO**，加载 `dataset_raw/`，类别对照本项目的
   `dataset/classes.txt`（raw / half-ripe / ripe）逐张画框；
4. 标完后按 8:2 划分并生成 data.yaml：

   ```bash
   python scripts/02_split_dataset.py --src dataset_raw --out dataset_real
   ```

   （`02_split_dataset.py` 会读取 `dataset_raw/classes.txt`，若没有则默认
   写入 raw/half-ripe/ripe 三类。）

> **半成熟最难，标注口径要统一**：half-ripe 是连续过渡里的一段，团队/个人要
> 先约定好"红晕占比多少算半熟"，并贯穿始终，否则模型会被矛盾标注带偏。

## 三、抽查标注是否正确（强烈建议，1 分钟）

数据质量直接决定训练效果。先把标注框画出来肉眼核对——**确认框住的确实是桃子、
成熟度标对了**（拿不准的可参考底色和红晕面积）：

```bash
python scripts/07_check_labels.py --data dataset_real --split train --n 16
```

执行后会生成一张拼图：`dataset_real/_label_check_train.jpg`，打开看每个框的
类别名和位置对不对。若发现大量错标或半熟/熟混淆，回到第二节修正后重做。

> 注：`07_check_labels.py` 从 `data.yaml` 读类别名，**类别无关**，三类两类都适用。

---

## 四、用真实数据训练模型

核心命令是 `03_train.py`，关键是把 `--data` 指向**真实数据集**的 yaml。
下面按你的机器三选一。

### 4.1 Mac（Apple 芯片 M1/M2/M3…）

```bash
python scripts/03_train.py --data dataset_real/data.yaml --device mps --batch 32 --epochs 100
```

> 16GB+ 统一内存跑 yolov8n + batch32 很从容；若提示内存紧张，把 `--batch` 降到
> 16，或加 `--cache False`。

### 4.2 Windows / Linux + NVIDIA 显卡

```bash
python scripts/03_train.py --data dataset_real/data.yaml --device 0 --batch 64 --epochs 150
```

> `--device 0` 表示第 0 块 GPU。显存小（如 6GB）就把 `--batch` 降到 16 或 8。

### 4.3 纯 CPU 机器（无独立显卡）

```bash
python scripts/03_train.py --data dataset_real/data.yaml --device cpu --batch 8 --epochs 40
```

> CPU 训练较慢，先用较少 epochs（如 40）跑通全流程，看效果满意再加大。

### 4.4 训练产物

训练结束后，最佳权重保存在：

```
runs/detect/fruit_sorter/weights/best.pt
```

这就是后续 GUI 和预测要用的模型。训练曲线、混淆矩阵等图表在
`runs/detect/fruit_sorter/` 目录下。

> 小贴士：训练时关注终端里的 `mAP50` 指标，越接近 1 越好。重点看
> **混淆矩阵**（`confusion_matrix.png`）里 half-ripe 那一行/列——若它和 raw、
> ripe 互相误判仍多，说明半熟样本不够或标注口径不统一（回到第二节）。

## 五、评估模型效果

用验证集量化模型表现（mAP、各类别准确率/召回率，并生成图表）：

```bash
# Mac
python scripts/04_eval.py --data dataset_real/data.yaml --weights runs/detect/fruit_sorter/weights/best.pt --device mps
# NVIDIA
python scripts/04_eval.py --data dataset_real/data.yaml --weights runs/detect/fruit_sorter/weights/best.pt --device 0
# CPU
python scripts/04_eval.py --data dataset_real/data.yaml --weights runs/detect/fruit_sorter/weights/best.pt --device cpu
```

输出示例：

```
mAP50    : 0.85
mAP50-95 : 0.62
precision: 0.83
recall   : 0.80
```

## 六、在真实图片上测试 / 启动识别系统

### 6.1 对一批真实照片批量预测

把你自己拍的桃子照片放进任意文件夹（如 `my_photos/`），然后：

```bash
python scripts/05_predict.py --weights runs/detect/fruit_sorter/weights/best.pt --source my_photos --conf 0.25
```

带框结果保存在 `runs/detect/predict*/`，逐张确认成熟度是否分对了。

### 6.2 启动 GUI 识别系统（产线模拟）

```bash
python src/main_gui.py --weights runs/detect/fruit_sorter/weights/best.pt --source my_photos
```

界面里：raw→未熟道、half-ripe→半熟道、ripe→成熟道；检测框颜色为
青绿/黄绿/红。"图像来源"选"文件夹轮播"即可用你的真实照片做演示。

> 只要训练时类别 id 与本项目一致（0-raw 1-half-ripe 2-ripe），
> GUI 和 detector **无需改任何代码**就能直接用新模型。

## 七、想要效果再上一个台阶（可选）

1. **混入你的实际场景照片**：现成数据集是网络通用照片。若你的分拣台有固定的
   背景、光照、相机角度，用手机拍几十张该场景的桃子，用 LabelImg 标注后放进
   `dataset_raw/`，跑 `python scripts/02_split_dataset.py` 合并训练。越贴近部署
   环境，效果越稳。

2. **半成熟持续加量**：这是本任务最难的一类。哪怕 raw/ripe 够了，也值得给
   half-ripe 多喂数据，并保证标注口径统一。

3. **换更大的模型**：`03_train.py` 默认用 yolov8n（最小最快）。追求精度可改用
   yolov8s/m：

   ```bash
   python scripts/03_train.py --data dataset_real/data.yaml --weights yolov8s.pt
   ```

## 八、常见问题速查

| 现象 | 原因 / 解决 |
|------|-------------|
| `data.yaml` 里 names 顺序和项目不一致 | YOLO 只认 id，按数据集导出的 yaml 为准训练即可；GUI 显示名在 `src/detector.py` 的 `CN_NAME` 调整 |
| Roboflow 导出选错格式 | 重新 Download，格式必须选 **YOLOv8** |
| half-ripe 和 raw/ripe 总混 | 半熟样本不够或标注口径不统一 → 回到第二节加量、统一口径，看混淆矩阵定位 |
| 训练报内存/显存不足 | 调小 `--batch`（如 16→8），或 `--cache False` |
| 真实图仍认不准 | 数据量不够或不贴合场景 → 回到第二、七节加量 |
| `mps`/`cuda` 报错 | 改 `--device cpu` 先跑通，再排查驱动 |
| 想先验证流程没数据 | 先用合成数据跑通：`python scripts/01_make_dataset.py` 再训练 |

## 九、全流程速览（以 Roboflow 数据 + Mac 为例）

```bash
cd ~/Desktop/fruit-sorting-yolo
python3 -m venv .venv && source .venv/bin/activate
pip install -U pip && pip install -r requirements.txt

# 1) 从 Roboflow Universe 下载 peach ripeness 数据集(YOLOv8 格式)，解压到 dataset_real/
#    并核对 data.yaml 的 names 顺序为 raw/half-ripe/ripe

# 2) 抽查 → 训练 → 评估
python scripts/07_check_labels.py --data dataset_real --split train --n 16
python scripts/03_train.py --data dataset_real/data.yaml --device mps --batch 32 --epochs 100
python scripts/04_eval.py  --data dataset_real/data.yaml --weights runs/detect/fruit_sorter/weights/best.pt --device mps

# 3) 在真实照片上验证
python scripts/05_predict.py --weights runs/detect/fruit_sorter/weights/best.pt --source my_photos
python src/main_gui.py       --weights runs/detect/fruit_sorter/weights/best.pt --source my_photos
```

> Windows/Linux+N卡把 `--device mps` 换成 `--device 0`；纯 CPU 换成
> `--device cpu` 并把 `--batch` 调到 8、`--epochs` 调到 40。
